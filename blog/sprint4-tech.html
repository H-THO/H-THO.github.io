Problem Solving Skills

Get Blocked

Sometimes getting blocked is unavoidable. It can be helpful to have a number of techniques or approaches to working through problems like this.

On sprint 4 I was blocked initially in the gradebook kata. I was unable to see how I was supposed to assign the scores correctly to the students within the gradebook. I had tried a number of things already - I'd taken the problem and broken it down into steps and expressed this in pseudocode. The main issue was in my proposed method to solve the issue - I had sort of misunderstood that there were much simpler ways to solve it.

My method was to assign the students by looping through the students, assigning the student objects to the gradebook, and then tried to add the scores to the students, while still in the for loop. To me coming from some use of Python - I figured I should be able to assign the scores using something like pythons dict(zip()), and of course doing so all within one loop would be the most 'elegant' way of solving the problem. I tried researching analogues of the dict(zip()) function implemented in javascript, which would've taken about 15 more lines of code. It also feels a bit hacky to work like this - not writing as the language of choice would dictate the style and methods for a solution.

I tried to force a solution, trying to explain via Rubber Duck this way but came up with nothing - I couldn't access the students using index notation as they were objects (e.g. keys aren't indexed as in an array). I was getting frustrated at this point, so i took a break to stretch and have a coffee. At this point I decided I'd try to find a different method of solution. After looking at the instructions again, I clicked.

I wasn't required to do this all in one step, in fact I didn't have to use such a complex method to assign the student objects or the scores! Once I realised this, I was able to assing them all correctly by just using object literal notation and accessing array indexes statically from here. This was a question that was quite simple, but I'd managed to decide it was really difficult.

Sometimes it can be really helpful to step back out of a problem and reassess the needs of the problem, and think a bit more laterally about how we could solve it. It was like my brain had gotten 'stuck' on one narrow possibility.

Elegant Solutions?

Sometimes, we're "lucky" and a solution pops straight out to us. I would't call this that elegant, but I can't say that many elegant solutions came to me during the sprint 4 Kata.

In the Super Fizzbuzz kata, I wrote pseudocode to solve the problem, identifying 3 cases - where we would alter the array value to a string. When the number was:
  1. A factor of 3
  2. A factor of 5
  3. A factor of both 3 and 5

I could see that condition 3 was actually just both conditions 1 and 2. It then hit me I could this using only two conditionals and working with a placeholder 'answer' string. See the code below: 

                
                        function super_fizzbuzz (arr) {
                            for (i=0; i < arr.length; i++){
                              fzbz = ""
                              if (arr[i] % 3 === 0){fzbz+="Fizz"}
                              if (arr[i] % 5 === 0){fzbz+="Buzz"} 
                              if (fzbz.length > 0){arr[i] = fzbz}
                            }
                          return arr
                        }

The process of solving that felt mostly unconcious and managed it way under the timebox allocation for the problem. I knew there were solutions for similar fizzbuzz challenges online but I wanted to have a go at it myself first. I'd knew I'd actually done a fizzbuzz challenge when doing a python codeacademy course a few years ago, so I was able to understand generally how I might solve the problem

I knew I had to use the modulo operator (%), so I read the docs on it and used the Rubber Duck method to understand how I'd implement the pseudocode. It felt quite seemless to solve in this way, and I managed to come up with a solution I thought was fairly elegant. It still felt a little cheaty as I had come across fizzbuzz before. I did have a lot more blocks on the later challenges though, I think I was definitely a bit "lucky" here.
