CSS Classes and IDs

Class Selectors

Class selectors allow us to specify css rules to be applied to HTML elements based on their element class. This allows us to access style rules use for commonly repeated styles or types of element within our website.

ID Selectors

ID Selectors behave a little differently to classes. A class allows reusability, while an ID is unique.

  • Each HTML element can only have one ID associated with it (while class selectors can select multiple classes).
  • Each web page can also have only one element with that specific ID.

ID Selectors and Specificity

ID selectors become useful as they have High Specificity. This means that they take precedence in the cascading behaviour of CSS. For an element that has both class and id rules applied to it will always have conflicting style rules dictated by the ID style rules.

Best Practices: Using Class and ID Selectors

When creating the basic blog structure for this course I found myself mostly finding it best to use classes and class selectors. A bit of research on the web suggests a number of differing viewpoints on best practice, with some even opting to not even use ID's at all. After reading a little discussion of best practices in CSS ID/Class usage on Stack Overflow I've come to appreciate that ID's only really have much utility to me as a way to manually override class behaviour for a single instance on a page, owing to their high specificity.

The accepted answer of the discussion linked suggest that if there is "more than one element which requires the same style, now or at any time in the future", then it should be implemented as a class.
